## 1. 프로젝트 개요

### 1.1 배경 및 목적

현재 인프라 구조도 작성 시 PowerPoint, Visio 등 범용 도구를 사용하고 있어 다음의 문제가 존재한다.

- 서버 상세 정보(OS 버전, DB 버전, IP, 호스트명 등)를 구조도 내에 체계적으로 표현하기 어려움
- 통신 포트 정보(DB 통신, RFC 통신 등)를 명확하게 표현할 수 없음
- 작성한 구조도를 범용 파일 형식으로 저장·공유하고, 다시 불러와 동일하게 복원·편집하는 것이 어려움

이 도구는 IT 인프라 담당자가 서버 구성 및 통신 흐름을 시각적으로 작성·편집·저장·복원할 수 있는 전용 웹 기반 편집기이다.

### 1.2 목표

- 서버 노드와 통신 연결선을 드래그&드롭으로 배치할 수 있는 인터페이스 제공
- 서버별 상세 메타데이터(OS, DB, SW 버전, IP, 호스트명 등)를 노드 내부에 입력 및 표시
- DB 통신, RFC 통신 등 포트 정보를 연결선에 표현
- 작성한 구조도를 HTML 파일로 저장하고 다시 불러와 동일하게 복원 및 재편집 가능
- 이미지(PNG/SVG) 캡처 및 내보내기 기능 제공

### 3.2 서버 노드

#### 3.2.1 노드 종류

다음의 서버 유형을 노드로 제공한다.

- 물리 서버 (Physical Server)
- 가상 머신 (VM)
- 데이터베이스 서버 (DB Server)
- 웹/애플리케이션 서버 (WAS/Web Server)
- 방화벽 (Firewall)
- 로드밸런서 (Load Balancer)
- 클라이언트 (Client / PC)
- 외부 시스템 (External System)
- 사용자 정의 노드 (Custom Node)

#### 3.2.2 노드 내부 표시 정보

각 서버 노드에 다음 정보를 입력하고 노드 내부에 표시한다.

| 필드명 | 설명 | 필수 여부 |
|--------|------|-----------|
| 서버명 (노드 타이틀) | 구조도 상 표시될 서버 이름 | 필수 |
| 호스트명 (Hostname) | 서버 호스트명 | 필수 |
| IP 주소 | 내부 IP (복수 입력 가능) | 필수 |
| OS 종류 및 버전 | 예: RHEL 8.6, Windows Server 2019 | 필수 |
| DB 종류 및 버전 | 예: Oracle 19c, MySQL 8.0 | 선택 |
| SW/미들웨어 버전 | 예: Tomcat 9.0, WebLogic 14.1 | 선택 |
| CPU / Memory 사양 | 예: 16 Core / 64 GB | 선택 |
| 역할 / 설명 | 해당 서버의 역할 요약 | 선택 |
| 운영 환경 | 운영(PRD) / 개발(DEV) / 스테이징(STG) | 선택 |
| 태그 | 사용자 정의 태그 | 선택 |

#### 3.2.3 노드 표시 옵션

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| F-10 | 노드 내부에 입력된 정보를 요약 또는 전체 표시 모드로 전환할 수 있다. | 필수 |
| F-11 | 노드 배경색, 테두리 색상을 사용자가 지정할 수 있다. | 권장 |

### 3.9 CSV 서버 목록 가져오기 (Import)

서버 정보를 CSV 파일로 일괄 입력하여 캔버스에 노드로 자동 생성할 수 있다.

#### 3.9.1 CSV 양식 다운로드

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| F-80 | 상단 툴바 또는 가져오기 패널에 **"CSV 양식 다운로드"** 버튼을 제공한다. | 필수 |
| F-81 | 다운로드되는 CSV 파일에는 헤더 행과 작성 예시 행(샘플 데이터 1~2건)이 포함되어야 한다. | 필수 |
| F-82 | CSV 헤더는 영문 컬럼명과 함께 한국어 설명 주석 행을 함께 제공한다. | 권장 |

**CSV 양식 컬럼 정의:**

| 컬럼명 (영문) | 한국어 설명 | 필수 여부 | 예시 값 |
|---------------|-------------|-----------|---------|
| `label` | 서버명 (노드 표시명) | 필수 | WAS 서버 #1 |
| `hostname` | 호스트명 | 필수 | was01.internal.com |
| `ip` | IP 주소 (복수 시 세미콜론 구분) | 필수 | 10.0.1.20;10.0.1.21 |
| `type` | 노드 유형 | 필수 | was / db / web / fw / lb / vm / physical / external / custom |
| `os` | OS 종류 및 버전 | 필수 | RHEL 8.6 |
| `db` | DB 종류 및 버전 | 선택 | Oracle 19c |
| `sw` | SW/미들웨어 버전 | 선택 | Tomcat 9.0.65 |
| `cpu_memory` | CPU / Memory 사양 | 선택 | 16 Core / 64 GB |
| `role` | 서버 역할 설명 | 선택 | 주문 처리 WAS |
| `env` | 운영 환경 | 선택 | PRD / DEV / STG |
| `zone` | 소속 네트워크 존 이름 | 선택 | Internal Network |
| `tags` | 사용자 태그 (세미콜론 구분) | 선택 | core;order |
| `color` | 노드 배경색 (HEX) | 선택 | #E3F2FD |

#### 3.9.2 CSV 가져오기 (Import)

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| F-85 | 상단 툴바 또는 가져오기 패널에서 CSV 파일을 업로드할 수 있다. | 필수 |
| F-86 | CSV 업로드 후 **미리보기 화면(테이블 형태)** 을 제공하여 데이터를 확인한 후 가져오기를 확정할 수 있다. | 필수 |
| F-87 | CSV의 각 행이 캔버스에 서버 노드로 자동 생성된다. | 필수 |
| F-88 | 노드의 초기 배치는 자동 정렬(격자 또는 계층형)로 배치되며, 이후 사용자가 수동으로 이동할 수 있다. | 필수 |
| F-89 | `zone` 컬럼 값이 동일한 노드는 같은 존(Zone) 그룹 박스 안에 자동 배치된다. | 권장 |
| F-90 | 필수 컬럼(`label`, `hostname`, `ip`, `type`, `os`) 누락 시 해당 행에 오류 표시를 하고 가져오기를 부분 허용한다. | 필수 |
| F-91 | 가져오기는 **신규 추가** 또는 **전체 교체** 중 선택할 수 있다. | 권장 |
| F-92 | 드래그&드롭으로 CSV 파일을 캔버스 영역에 올려도 가져오기가 실행된다. | 선택 |

#### 3.9.3 CSV 가져오기 흐름

```
[CSV 양식 다운로드]
        ↓
  Excel 등으로 서버 목록 작성
        ↓
[CSV 업로드 버튼] 또는 [캔버스에 파일 드래그&드롭]
        ↓
  미리보기 / 유효성 검사 화면
  - 오류 행 하이라이트 표시
  - 정상 행 건수 / 오류 행 건수 표시
        ↓
  [가져오기 확정] 클릭
        ↓
  캔버스에 서버 노드 자동 생성 및 배치
```

## 4. 비기능 요구사항
