## 1. 프로젝트 개요

### 1.1 배경 및 목적

현재 인프라 구조도 작성 시 PowerPoint, Visio 등 범용 도구를 사용하고 있어 다음의 문제가 존재한다.

- 서버 상세 정보(OS 버전, DB 버전, IP, 호스트명 등)를 구조도 내에 체계적으로 표현하기 어려움
- 통신 포트 정보(DB 통신, RFC 통신 등)를 명확하게 표현할 수 없음
- 서버 내 복수의 서비스(DB, 미들웨어 등)를 개별적으로 표현하고 서비스 간 연결을 시각화하기 어려움
- 작성한 구조도를 범용 파일 형식으로 저장·공유하고, 다시 불러와 동일하게 복원·편집하는 것이 어려움

이 도구는 IT 인프라 담당자가 서버 구성 및 통신 흐름을 시각적으로 작성·편집·저장·복원할 수 있는 전용 웹 기반 편집기이다.

### 1.2 목표

- 서버 노드와 통신 연결선을 드래그&드롭으로 배치할 수 있는 인터페이스 제공
- 서버별 상세 메타데이터(OS, 서비스, IP, 호스트명 등)를 노드 내부에 테이블형으로 입력 및 표시
- 서버 내 개별 서비스(DB/미들웨어/애플리케이션) 단위의 연결선 표현
- 컨테이너 노드(Zone/방화벽/VM)로 인프라 계층 구조 표현
- 직각(orthogonal) 연결선이 서버 노드를 자동 회피하여 라우팅
- 작성한 구조도를 HTML 파일로 저장하고 다시 불러와 동일하게 복원 및 재편집 가능
- 이미지(PNG/SVG) 캡처 및 내보내기 기능 제공

### 3.2 서버 노드

#### 3.2.1 노드 종류

**서버 노드 (일반):**

- 물리 서버 (Physical Server)
- 데이터베이스 서버 (DB Server)
- 웹/애플리케이션 서버 (WAS/Web Server)
- 로드밸런서 (Load Balancer)
- 클라이언트 (Client / PC)
- 외부 시스템 (External System)
- 사용자 정의 노드 (Custom Node)

**컨테이너 노드 (자식 노드 포함 가능):**

- 네트워크 존 (Zone) — 가장 바깥 레벨
- 방화벽 (Firewall) — 서버 그룹을 감싸는 보안 경계
- 가상 머신 (VM) — 서버를 호스팅하는 하이퍼바이저

> **중첩 규칙:** Zone > Firewall > VM > Server (최대 4단계)  
> 드래그로 자식을 컨테이너 안/밖으로 이동 가능.  
> 컨테이너 삭제 시 자식 노드는 자동 해제.

#### 3.2.2 노드 내부 표시 정보

각 서버 노드에 다음 정보를 입력하고 노드 내부에 **테이블형 레이아웃**으로 표시한다.

| 필드명 | 설명 | 필수 여부 |
|--------|------|-----------|
| 서버명 (노드 타이틀) | 구조도 상 표시될 서버 이름 (진한 색상 타이틀 바) | 필수 |
| 호스트명 (Hostname) | 서버 호스트명 | 필수 |
| IP 주소 | 내부 IP (복수 입력 가능, 회색 배경 행) | 필수 |
| OS 종류 및 버전 | 예: RHEL 8.6, SUSE 12 SP3 (노란색 배경, 맨 아래 행) | 필수 |
| 서비스 목록 | DB/미들웨어/애플리케이션 (각 서비스별 개별 행) | 선택 |
| CPU / Memory 사양 | 예: 16 Core / 64 GB | 선택 |
| 역할 / 설명 | 해당 서버의 역할 요약 | 선택 |
| 운영 환경 | 운영(PRD) / 개발(DEV) / 스테이징(STG) | 선택 |
| 태그 | 사용자 정의 태그 | 선택 |

#### 3.2.3 서비스 (Services)

각 서버 노드에 복수의 서비스를 등록할 수 있다. 서비스는 노드 내부에 개별 행으로 표시된다.

| 필드명 | 설명 | 예시 |
|--------|------|------|
| 유형 (type) | DB / 미들웨어 / 애플리케이션 / 기타 | DB |
| 이름 (name) | 서비스명 | Oracle 19c |
| 포트 (port) | 리스닝 포트 | 1521 |
| SID | 서비스 인스턴스 식별자 | ORCL |
| 설명 (description) | 부가 설명 | 메인 운영 DB |

> **노드 카드 표시:** SID가 있으면 `SID (이름) :포트` 형식 (예: `ORCL (Oracle 19c) :1521`)  
> **행 색상 규칙:** DB 서비스 = 연노랑 배경, 기타 서비스 = 흰색 배경

| F-89 | `zone` 컬럼 값이 동일한 노드는 같은 존 그룹 박스 안에 자동 배치된다. | 권장 | ✅ 완료 |
| F-90 | 필수 컬럼 누락 시 해당 행에 오류 표시를 하고 가져오기를 부분 허용한다. | 필수 | ✅ 완료 |
| F-91 | 가져오기는 **신규 추가** 또는 **전체 교체** 중 선택할 수 있다. | 권장 | ✅ 완료 |

## 10. 용어 정의

| 용어 | 정의 |
|------|------|
| 노드 | 캔버스 위에 배치되는 서버 또는 시스템을 나타내는 요소 |
| 컨테이너 노드 | 다른 노드를 포함할 수 있는 Zone/방화벽/VM 노드 |
| 서비스 (Service) | 서버 내에서 구동되는 개별 DB/미들웨어/애플리케이션 인스턴스 |
| SID | Service Instance Identifier. DB 또는 SAP 인스턴스 식별자 |
| 엣지 (Edge) | 노드 간 또는 서비스 간의 통신 연결을 나타내는 선 |
| 핸들 (Handle) | 노드 또는 서비스 행의 연결점. 드래그하여 엣지 생성 |
| 존 (Zone) | 네트워크 영역을 나타내는 컨테이너 노드 |
| RFC 통신 | SAP 시스템 간 Remote Function Call 방식의 통신 |
| 오프라인 동작 | 인터넷 연결 없이 로컬 파일로만 동작하는 것 |
